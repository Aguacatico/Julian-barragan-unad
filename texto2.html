Calidad De Software

La calidad de software actualmente se trabaja sobre las normas ISO/25000 las cuales se refieren a un lineamiento que funciona como guía para asegurar la calidad del software que se esta produciendo

La calidad del software se debe garantizar por todo el ciclo de vida que tiene el software el cual es el siguiente:

  • Definición de objetivos
  • Análisis de los requisitos y su viabilidad
  • Diseño general
  • Diseño en detalle
  • Programación
  • Prueba de unidad
  • Integración
  • Prueba beta o validación
  • Documentación
  • Implementación
  • Mantenimiento

Desde el levantamiento de información se debe garantizar que el software va a poseer seguimiento de calidad por parte de los desarrolladores de este mismo para así al finalizar el ciclo de vida se pueda entregar un producto acercándose a la perfecta calidad la cual es solicitada por el usuario final el cual debe recibir el software cumpliendo las expectativas de calidad
